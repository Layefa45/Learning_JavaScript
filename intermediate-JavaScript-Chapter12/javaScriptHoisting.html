JAVASCRIPT HOISTING

Hoisting is the principle of moving declarations of variables to the top of the scope in which they are defined.
This allows you to do things that you cannot do in many other languages. 

    var x;
    x = 5;
    console.log(x); // OUTPUT: 5

But thanks to hoisting, so does this; 

            
    x = 5;
    console.log(x); //OUTPUT: 5
    var x;

If you try to do this with 'let', you will get a 'ReferenceError'. This is why it is better to use 'let'. Because this behaviour is very hard to read.
And sometimes very unpredictable, and you really don't need it.

The reason why this happens is that the JavaScript interpreter moves all the 'var' declarations to the top of the file before processing the file.
This is why you get a result of 'undefined' if you use it before it has been declared.
It was designed this way for memory allocation, but the side effects are undesirable.
However there is a way to turn off this behaviour, using strict mode.
