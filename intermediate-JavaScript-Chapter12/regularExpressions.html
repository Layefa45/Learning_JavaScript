Regular Expressions

Regular Expressions are simply ways to describe text patterns. They are also called 'Regex'.

Regex is not only useful for finding strings, but can also be used for replacing strings. The below is an email validation check in Regex.

/([a-zA-Z0-9._-]+@[a-zA-Z0-9._-]\.[a-zA-Z0-9._-]+)/g

Check the console out to see outputs of code.

let text = 'I love Javascript!'; console.log(text.match(/javascript/)); // OUTPUT NULL

let text = 'I love Javascript!'; console.log(text.match(/javascript/i)); // CHECK CONSOLE (i) MAKES IT CASE INSENSITIVE.

let text2 = 'I love Javascript!'; console.log(text2.match(/javascript|nodejs|react/i)); // THIS RETURNS EITHER Javascript, NODEJS OR REACT WHEN IT FINDS A FIRST MATCH.

GROUPS

There are many reasons to group your regex. Whenever you want to match a group of characters you can surround them with parentheses.

let text = 'I love JavaScript'; console.log(text.match(/(love|dislike)/s(javascript|spiders)/gi));

EMAIL VALLIDATION

In order to create a regex pattern, we need to be able to describe the patterns with words first.

Email addresses consists of five parts, in the form '[name]@[domain].[extension]'.

The parts explained:

'Name'
One or more alphanumerical characters, underscores, dashes or dots.
'@'
Literal Character.
'Domain'
One or more alphanumerical characters, underscores, dashes or dots.
'.'
Literal dot.
'Extension'
One or more alphanumerical characters, underscores, dashes or dots.

The steps for the regex:

  • [a-zA-Z0-9._-]+
  • @
  • [a-zA-Z0-9._-]+
  • \.
  • [a-zA-Z0-9._-]+

Putting it all together:
/([a-zA-Z0-9._-]+@[a-zA-Z0-9._-]+\.[a-zA-Z0-9._-]+)/g

    let emailPattern = /([a-zA-Z0-9._-]+@[a-zA-Z0-9._-]+\.[a-zA-Z0-9._-]+)/g;
    let validEmail = 'maaike_1234@email.com';
    let invalidEmail = 'maaike@mail@.com';
    console.log(validEmail.match(emailPattern));
    console.log(invalidEmail.match(emailPattern));
